# 🪄 Лафка у Оливандера - e-commerce магазин волшебных товаров

Полнофункциональное веб-приложение интернет-магазина волшебных товаров на основе Node.js/Express с REST API, JSON базой данных и Swagger документацией.

## 📖 Описание проекта

Это учебный проект, демонстрирующий:
- ✅ REST API архитектуру с выделением бизнес-контекстов
- ✅ JSON-based персистентную базу данных (нормализация по 3НФ)
- ✅ Полную интеграцию Swagger/OpenAPI с примерами
- ✅ CRUD операции (Create, Read, Update, Delete)
- ✅ Валидацию данных на серверной стороне
- ✅ Фронтенд интеграцию с vanilla JavaScript
- ✅ Админ-панель для управления товарами, заказами и услугами
- ✅ 12 примеров товаров в каталоге
- ✅ Примеры заказов и бронирований в базе данных

## 🚀 Быстрый старт на Windows

### Требования
- **Node.js** версия 14+ ([скачать с nodejs.org](https://nodejs.org/)) - LTS версия рекомендуется
- **npm** (идет вместе с Node.js)

### Проверка установки
Откройте PowerShell и выполните:
```powershell
node --version
npm --version
```
Если команды не найдены, перезагрузите компьютер или добавьте Node.js в PATH.

### Установка и запуск (3 шага)

1. **Перейти в папку проекта** (используйте кавычки!):
   ```powershell
   cd "c:\Users\yaksha\Desktop\project\project\HarryPotter"
   ```

2. **Установить зависимости**:
   ```powershell
   npm install
   ```

3. **Запустить сервер**:
   ```powershell
   npm start
   ```

✅ Сервер запустится на **http://localhost:3000**

## 🔗 API Endpoints

### Товары (Products)
| Метод | Endpoint | Описание |
|-------|----------|---------|
| GET | `/api/products` | Получить все товары |

|-------|----------|---------|
| GET | `/api/bookings` | Получить все бронирования (админ) |

Проект использует **JSON базу данных** (database.json) с 3 таблицами:

### Товары (12 примеров)
- Волшебные палочки (1200-1500 ₽)
- Метлы для полетов (4500-8500 ₽)
- Зелья и эликсиры (150-950 ₽)
- Аксессуары и книги (300-600 ₽)

### Заказы
- Примеры с разными статусами (processing, completed)
- Полная история покупок

### Бронирования
- Примеры записей на мастер-классы
- Статусы: pending, confirmed, cancelled

## 📊 Архитектура

Проект организован по **4 бизнес-контекстам:**

### 1️⃣ Product Management (Управление товарами)
- Просмотр каталога
- Получение деталей товара
- Проверка наличия на складе
- **Файлы:** catalog/*, server.js (GET /api/products)

### 2️⃣ Order Processing (Обработка заказов)
- Оформление заказа
- Валидация и проверка склада
- История заказов
- **Файлы:** order/checkout.html, server.js (POST /api/orders)

### 3️⃣ Service Booking (Бронирование услуг)
- Запись на мастер-классы
- Управление датами
- Отслеживание бронирований
- **Файлы:** service/book.html, server.js (POST /api/bookings)

### 4️⃣ Administration (Администрирование)
- Просмотр всех заказов
- Управление статусами
- Управление товарами
- **Файлы:** admin/*, server.js (GET endpoints)

## 🧪 Примеры использования API

### Получить список товаров
```bash
curl http://localhost:3000/api/products
```

### Оформить заказ
```bash
curl -X POST http://localhost:3000/api/orders \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Иван Петров",
    "email": "ivan@example.com",
    "phone": "+7-900-123-45-67",
    "address": "ул. Волшебная, д. 5",
    "productId": 1,
    "quantity": 1
  }'
```

### Забронировать услугу
```bash
curl -X POST http://localhost:3000/api/bookings \
  -H "Content-Type: application/json" \

    "phone": "+7-902-111-22-33",
    "email": "alex@example.com",
    "serviceType": "Мастер-класс по палочкам",
    "date": "2025-12-28"
  }'
```

## 🔐 Безопасность (важно для прода)

⚠️ **Текущая версия - учебный проект!**

Для использования в production:
- ❌ Нет реальной аутентификации (админ открыта)
- ❌ Базовая валидация (без защиты от XSS)
- ❌ CORS разрешен для всех источников
- ❌ JSON БД не имеет ACID гарантий

**Рекомендации для продакшена:**
- ✅ Использовать PostgreSQL/MongoDB вместо JSON
- ✅ Добавить JWT аутентификацию
- ✅ Имплементировать input sanitization
- ✅ Использовать HTTPS
- ✅ Добавить rate limiting
